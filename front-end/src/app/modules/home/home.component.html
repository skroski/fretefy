Documentação do Desafio

Introdução

Primeiramente, gostaria de expressar minha gratidão pela oportunidade de participar deste desafio. A experiência foi enriquecedora e, com base nas exigências, acredito que consegui entregar uma solução robusta, especialmente no Front-End. Investi também na melhoria da interface do usuário (UI) e na experiência do usuário (UX), com o objetivo de oferecer um resultado visualmente atraente e funcional.

Enfrentei um desafio inicial relacionado à versão 10.2 do Angular, mas após realizar ajustes necessários, como a atualização da versão do Node (para a 16) e do Angular (para a 10.2) utilizando o nvm, consegui fazer a aplicação funcionar conforme esperado.

Metodologia

Coleta de Dados

Para obter as informações necessárias sobre as regiões do Brasil, estados e municípios, utilizei a API do IBGE. Para garantir uma manipulação eficiente dos dados, implementei tipagem forte na pasta models, facilitando o processo de integração e validação.

Uso do Angular

A aplicação foi desenvolvida utilizando a versão 10 do Angular. Durante o desenvolvimento, utilizei diversas funcionalidades, como Input() e Output(), ngClass, serviços, modelos, validadores e componentes.

  • Organizei as pastas de acordo com as melhores práticas recomendadas, embora, com mais tempo, eu possa adaptar a estrutura à documentação oficial do Angular.
  • A maior parte dos componentes foi construída com HTML e CSS nativos, incluindo o uso de SASS para um design mais flexível e escalável.
  • Considerando a natureza da vaga (Designer de Produto), adotei cores e elementos visuais inspirados na identidade visual da Fretefy, para criar uma solução alinhada com o sistema de design da empresa.

Resultados

Desafios e Soluções

O processo de adaptação às tecnologias exigidas foi desafiador, mas consegui integrar frameworks como Tailwind, Bootstrap, DaisyUI, PrimeNG e Angular Material com Material Design, o que proporcionou uma experiência de usuário agradável e consistente.

A validação de municípios ainda não foi implementada, mas planejo realizar esse ajuste em breve, com o objetivo de evitar o cadastro duplicado de cidades.

  • Foi implementado um CRUD completo com localStorage, com a adição de um popover nativo. Ao passar o mouse sobre o nome da tabela, uma descrição detalhada é exibida no popover.
  • Embora o back-end não tenha sido integrado, utilizei a API do IBGE e o localStorage para persistir dados temporários. Para uma versão futura, seria interessante integrar com bancos de dados como MongoDB ou Supabase e adicionar novas funcionalidades.

Design

Ao analisar as cores e fontes do site da Fretefy, criei uma adaptação visual que reflete a identidade visual da empresa. Essa abordagem inicial pode ser ajustada conforme a necessidade, para garantir uma maior consistência com o sistema de design utilizado.

  • A tipografia utilizada segue a mesma família visual do site da Fretefy, garantindo coesão no design.
  • O botão de exportação para Excel foi implementado utilizando as cores do software, proporcionando uma integração visual mais harmoniosa.
  • Popovers nativos foram criados para as funcionalidades de edição e exclusão, com tratamento especial para o dropbox.

Conclusão

Embora tenha enfrentado alguns desafios, pude realizar o desenvolvimento com calma, sempre buscando uma solução bem estruturada e de qualidade. A organização do código Angular também foi um ponto importante, garantindo um bom equilíbrio entre a parte visual e a funcionalidade da aplicação.

Próximos Passos

Ainda planejo finalizar a validação de municípios e implementar melhorias adicionais, como a integração com back-end e a otimização de funcionalidades, entregando a versão final assim que todos os ajustes forem concluídos.
